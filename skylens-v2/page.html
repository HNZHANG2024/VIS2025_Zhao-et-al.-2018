Theoretical Background

在多维数据分析中，用户往往面临着基于相互冲突的标准进行选择的困难。Skyline 查询（又称 Pareto 优化）通过过滤掉那些被其他点“支配”（Dominated）的数据点，提取出最优解集合。

然而，随着维度的增加，Skyline 集合往往依然庞大且难以解释。本项目复现并扩展了 Zhao et al. (2018) 提出的 SkyLens 系统。该系统通过多视图协调（Coordinated Multiple Views）的方式，不仅展示了 Skyline 集合，还通过量化“支配评分”（Dominance Score），直观地解释了为什么某个点属于 Skyline，以及它在哪些子空间中表现优异。

Implementation Details

本项目不仅仅是简单的复现，还针对现代浏览器环境进行了性能优化。核心架构采用 React 19 组件化设计，利用 Vite 进行构建，并使用 TypeScript 保证代码健Tb性。

  • WebGPU Acceleration: 针对原算法 O(n²) 的计算复杂度，我实现了一个基于 WebGPU 的并行计算模块。通过 Compute Shader 并行处理 Dominance 比较，在大数据量下实现了流畅的交互体验（这是原论文未涉及的改进）。
  • Custom Visualizations (D3.js): 所有的视图组件（ProjectionView, RadarChart, TabularView）均使用 D3.js 定制开发，而非调用现成图表库。例如，ComparisonView 中的字形（Glyph）通过 SVG 路径动态生成，能够同时编码属性值和支配关系。
  • Interactive Linking: 实现了全局状态管理。在 Tabular View 中 Brush 筛选的数据，会实时更新 Projection View 的 t-SNE 布局；鼠标悬停（Hover）操作会触发全视图的高亮联动。

Application Features

系统复现了论文中的三个核心视图，并保持了高度的联动性：

  • Projection View: 使用降WK算法将高维 Skyline 点映射到 2D 平面。点的颜色编码了其 Dominance Score，外围的环形图展示了关键属性的数值分布。
  • Tabular View: 利用“发散条形图”（Diverging Bar Chart）展示数据详情。每一行代表一个数据点，蓝色/红色条形直观地反映了该点相对于整体平均水平的优劣。
  • Comparison View: 专门用于深度对比的视图。用户选中感兴趣的点后，系统会生成雷达图与饼图的组合 Glyph，直观展示两点之间的支配强度（Dominance Strength）和差异。
